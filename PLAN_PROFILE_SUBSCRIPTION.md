# 🗺️ ПЛАН ВЫПОЛНЕНИЯ

**Задача:** Улучшение страницы профиля — подписка, тарифы, оплата (доработка)

**Тип:** Исправление дефектов в «завершённой» задаче

**Стратегия:** Основная UI-работа сделана, но найдены 3 критичных и 2 средних проблемы, из-за которых функциональность не работает корректно. Исправляем точечно, не трогая рабочий UI.

---

## 🔍 ОБНАРУЖЕННЫЕ ПРОБЛЕМЫ

| # | Проблема | Серьёзность | Где |
|---|----------|-------------|-----|
| **P1** | **Deep link оплаты не работает.** Кнопки в Mini App вызывают `openTelegramLink('t.me/BOT?start=subscribe_stars')` и `t.me/BOT?start=subscribe`, но обработчик `/start` в боте НЕ обрабатывает эти payload-ы — он проверяет только `ref_*`. Результат: пользователь нажимает «Оплатить» → попадает в бота → видит обычное приветствие вместо инвойса. **Flow оплаты из Mini App полностью нерабочий.** | Критическая | `packages/bot/src/commands/start.ts` |
| **P2** | **Сумма оплаты: 299 kopecks = 2.99 ₽, а не 299 ₽.** `SUBSCRIPTION_PRICE = 299` с комментарием «smallest currency unit». Для RUB smallest unit = копейка. 299 копеек = 2.99 ₽. Но в UI отображается «299 ₽ / 30 дней». Либо сумма в коде должна быть `29900`, либо текст в UI неверный. | Критическая | `packages/bot/src/payments/payments.ts`, i18n |
| **P3** | **Достижения — захардкоженные русские строки.** `ACHIEVEMENT_META` содержит `nameKey: 'Первое задание'`, `'3 дня подряд'` и т.д. — это реальный текст, а не i18n-ключи. Достижения ВСЕГДА отображаются на русском, независимо от локали пользователя. | Средняя | `packages/webapp/src/pages/Profile.tsx` |
| **P4** | **Тексты инвойсов и ответов бота захардкожены на русском.** Invoice description, сообщения об успешной оплате/ошибке — всё на русском, не используют i18n. | Средняя | `packages/bot/src/payments/payments.ts`, `packages/bot/src/payments/stars.ts` |
| **P5** | **Ошибки профиля проглатываются.** `.catch(() => {})` в `useEffect` — если один из трёх запросов (`getAchievements`, `getProgress`, `getMe`) падает, пользователь видит пустую страницу без объяснений. | Низкая | `packages/webapp/src/pages/Profile.tsx` |

|---|-----------|-------------------|------------|------------|
| 1.1 | В `packages/bot/src/commands/start.ts` — добавить обработку deep link параметров `subscribe` и `subscribe_stars`. Если `payload === 'subscribe'` → отправить инвойс через Telegram Payments (тот же код, что в `payments.ts` для `/subscribe` команды). Если `payload === 'subscribe_stars'` → отправить Stars-инвойс. Для DRY — вынести генерацию инвойсов в общие функции, вызываемые и из команд, и из start handler. | `packages/bot/src/commands/start.ts`, `packages/bot/src/payments/payments.ts`, `packages/bot/src/payments/stars.ts` | — | Нажатие кнопки «Оплатить Stars ⭐» в Mini App → переход в бота → бот отправляет Stars-инвойс. Аналогично для карты. |
| 1.2 | Добавить локализацию в start handler для subscribe-параметров: определять язык из `ctx.from?.language_code`, отвечать на нужном языке. | `packages/bot/src/commands/start.ts` | 1.1 | Инвойс отправляется с текстом на языке пользователя |

**Definition of done:** Полный flow: Mini App → кнопка оплаты → бот → инвойс → оплата → подписка активирована.

## ⚠️ РИСКИ ПЛАНА

| Риск | На каком шаге | Влияние | Митигация |
|------|---------------|---------|-----------|
| Вынос функций генерации инвойса может сломать регистрацию handlers | 1.1 | Высокое | Не менять регистрацию команд `/subscribe` и `/subscribe_stars` — они должны продолжать работать. Добавить вызовы тех же функций из start handler |
| Неверное значение `amount` для RUB в Telegram Payments API | 2.1 | Критическое | Проверить на тестовом провайдере перед продом. Telegram API документация: amount in smallest units (для RUB = kopecks) |
| `openTelegramLink` может не работать в некоторых версиях Telegram | 1.1 | Среднее | Альтернатива: `window.open('tg://resolve?...')`. Но `openTelegramLink` — официальный метод SDK |

## 🛡️ СТРАТЕГИЯ БЕЗОПАСНОСТИ

- Команды `/subscribe` и `/subscribe_stars` — продолжают работать как прежде (добавление в start НЕ убирает их)
- UI подписки в Profile.tsx — менять только `ACHIEVEMENT_META` и обработку ошибок, не трогать секцию подписки
- Изменение суммы (29900 vs 299) — **тестировать на тестовом платёжном провайдере** перед деплоем

**Не должно быть затронуто:**
- Логика активации подписки через API (`/api/internal/activate-subscription`)
- Обработка `pre_checkout_query` и `successful_payment`
- UI секции подписки (бейдж, таблица, кнопки)

## ❓ ОТКРЫТЫЕ ВОПРОСЫ

| Вопрос | Рабочая гипотеза |
|--------|------------------|
| Сумма 299 ₽ — kopecks или рубли? | 29900 kopecks = 299 ₽. Текущий код (299) — ошибка, нужно 29900 |
| Нужно ли закрывать Mini App при переходе к оплате? | Нет, `openTelegramLink` откроет чат с ботом, Mini App останется в фоне — это нормальное поведение |
| Работает ли `/subscribe` как прямая команда бота? | Да, регистрация `bot.command('subscribe')` сохраняется. Но пользователь может дойти до оплаты и через start deep link, и через команду |
