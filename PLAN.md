# 🗺️ ПЛАН ВЫПОЛНЕНИЯ

**Задача:** Telegram-платформа для тренировки мозга с социальными механиками (Brainify)

**Тип:** Новая функциональность (greenfield-проект)

**Стратегия:** Послойная сборка — сначала инфраструктура и данные, потом ядро (задания + бот), затем социальные механики, затем монетизация. Каждый этап даёт работающий инкремент. Mini App и бот развиваются параллельно после появления общего бэкенда.

---

## 🔧 РЕШЕНИЯ ПО СТЕКУ

| Вопрос | Решение | Обоснование |
|--------|---------|-------------|
| Хостинг | VPS (свой) | Есть VPS |
| Бэкенд | Node.js + TypeScript + Fastify | Быстрый, легковесный, отличная поддержка TypeScript |
| ORM | Drizzle ORM | Лёгкий, типобезопасный, близок к SQL (легко читать и отлаживать), минимальный overhead, простые миграции |
| БД | PostgreSQL + Redis | PostgreSQL — основное хранилище, Redis — кэш, очереди, счётчики лимитов |
| Фронтенд Mini App | Preact + Vite | Минимальный бандл (критично для Telegram WebView) |
| Фреймворк бота | grammY | Активная поддержка, отличная типизация |
| Языки | Русский + Английский | Определяется из Telegram, с возможностью переключения |

## ⚠️ РИСКИ ПЛАНА

## 🛡️ СТРАТЕГИЯ БЕЗОПАСНОСТИ

- **Каждый этап — работающий инкремент.** Можно остановиться на любом и иметь рабочий продукт.
- **Валидация Telegram initData (HMAC-SHA256)** — обязательна с первого дня.
- **Подписка реализуется последней** (этап 6) — до этого всё работает без ограничений.
- **Каждый этап — отдельная ветка/PR.** Откат — целиком.
- **Rate limiting** на API с первого дня.
- **Секреты** — только через переменные окружения, никогда в коде.

**Не должно быть затронуто:**
- Flow заданий не должен ломаться при добавлении социальных механик
- Работа без подписки (trial/free) должна сохраняться при добавлении платежей
