### ⚠️ РИСКИ РЕГРЕССИИ

- **`GET /api/tasks` теперь использует `optionalAuthMiddleware`** — подтверждён как безопасный: гостевой доступ работает (возвращает базовые данные без рекомендаций), для авторизованных добавляются `recommendedDifficulty` и `lastDifficulty`.
- **Сигнатура `onSelect` в `TaskCatalog`** изменена с `(taskId, taskType)` на `(taskId, taskType, difficulty)` — `App.tsx` обновлён. Все вызовы (каталог, daily challenge, training) передают 3 аргумента.
- **Дуэли** — `difficulty` в `PlayState` для дуэлей зафиксирован как `1`, дуэльный flow не затронут.
- **CSS self-reference pattern** (`--tg-theme-bg-color: var(--tg-theme-bg-color, #ffffff)`) — преexisting issue, не введено в этих изменениях. В TMA-окружении Telegram SDK инжектит переменные через inline-стили, так что fallback работает. В веб-версии за пределами TMA — потенциальная проблема. Стоит проверить визуально.
- **`onboardingDone` сбрасывается при навигации** — используется `useState(false)`, при перезагрузке страницы онбординг будет запрошен повторно, но API вернёт 409 (onboarding already completed). Корректное поведение: `App.tsx` проверяет `user.onboardingCompleted === false`, поэтому после reload `getMe()` вернёт `onboardingCompleted: true` и онбординг не покажется.
