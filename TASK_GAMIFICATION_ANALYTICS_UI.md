### 📋 ПОСТАНОВКА ЗАДАЧИ

**Название:** Геймификация, аналитика уровня пользователя и UI/UX-редизайн Brainify

**Цель:** Повысить вовлечённость и retention пользователей за счёт: (1) разнообразной сложности заданий и дополнительных игровых механик, (2) прозрачной аналитики когнитивного уровня с элементом социального сравнения и ценностного предложения подписки, (3) качественного, адаптивного и интуитивно понятного интерфейса. Конечная метрика — пользователь понимает свой уровень, видит путь роста, хочет продолжать заниматься и готов платить за подписку.

**Проблема:** Текущий продукт функционально работает, но не удерживает пользователя:
1. **Однообразность геймплея.** Сложность задаётся параметром `difficulty: 1-5` в генераторах, но UI не даёт пользователю выбирать или повышать уровень сложности. Все запускают задания с `difficulty: 1` (захардкожено по умолчанию). Нет прогрессии, нет челленджей, нет причины возвращаться.
2. **Отсутствие аналитики уровня.** Страница Progress показывает только streak, дневной бар-чарт баллов и количество попыток по категориям. Нет: оценки когнитивного уровня, динамики по отдельным навыкам, сравнения с другими пользователями, рекомендаций по развитию.
3. **Непонятный и некрасивый UI.** CSS — 100 строк в `global.css`, стили через inline-стили и CSS-переменные Telegram (`--tg-theme-*`). В веб-версии (не Telegram) темы выглядят бедно. Нет адаптации под разные размеры экрана (нет media queries). Кнопка «Войти через Telegram» — виджет-iframe, непонятный для пользователя на веб-сайте.

**Контекст:**
- Monorepo: `packages/api` (Fastify), `packages/webapp` (Preact + Vite), `packages/shared` (генераторы, типы, i18n), `packages/bot` (grammY)
- 7 типов заданий: schulte, sequence_memory, arithmetic, stroop, number_series, memory_pairs, pattern_search, по 4 категориям (memory, attention, logic, speed)
- Каждый генератор уже поддерживает `difficulty: 1-5` с масштабированием параметров (размер сетки, количество задач, диапазон чисел)
- API endpoint `POST /api/tasks/:id/start` принимает `difficulty` в body, валидирует 1-5
- БД хранит `difficulty` в таблицах `task_attempts` и `task_sessions`
- Модель подписки: trial (7 дней) → free (3 задания/день) → active (полный доступ)
- Авторизация: TMA (initData) / Web (JWT через Telegram Login Widget) / Guest
- Бюджет бандла webapp: ≤ 200 КБ gzip
- Stack: TypeScript, Preact 10.x, Vite 6.x, Fastify 5.x, Drizzle ORM, PostgreSQL 16, Redis 7

### 🎯 ГРАНИЦЫ (SCOPE)

**Входит:**

**Блок 1 — Уровни сложности и геймификация:**
- UI выбора сложности перед запуском задания (1-5 или именованные уровни: начинающий / средний / продвинутый / эксперт / мастер)
- Адаптивная сложность: рекомендация следующего уровня на основе результатов (если score > 80% — предложить повысить)
- Ежедневные челленджи / задание дня (выбранное задание повышенной сложности с бонусными очками)
- Серия заданий (мини-сессия из 3-5 заданий разных типов с общим результатом)

**Блок 2 — Аналитика и оценка уровня:**
- Первичное тестирование (онбординг-тест): набор заданий для определения базового уровня когнитивных навыков пользователя
- Когнитивный профиль: radar-chart по 4 категориям (memory, attention, logic, speed) с числовым рейтингом
- Перцентильное сравнение: «ваш результат выше X% пользователей вашего возраста» (требует сбор возрастной группы)
- Рекомендация пути развития: «чтобы достичь уровня 95%, занимайтесь 3-5 месяцев на нашем тренажёре»
- Динамика прогресса по каждой категории (графики изменения уровня за неделю / месяц)
- API endpoint для расчёта перцентилей на основе агрегированных данных `task_attempts`

**Блок 3 — UI/UX-редизайн:**
- Проработка собственной цветовой темы для веб-версии (light + dark), не завязанной только на `--tg-theme-*`
- Responsive design: адаптация под мобильные (320-480px), планшеты (768px), десктоп (1024px+)
- Переработка LoginScreen: понятная кнопка входа через Telegram, объяснение процесса, визуальная привлекательность
- Переработка каталога заданий: карточки с визуальным отображением сложности, категории, прогресса
- Переработка экрана результата: детальная статистика, сравнение с прошлым результатом, анимации
- Переработка навигации: улучшение NavBar для разных экранов
- Микроанимации и визуальная обратная связь (transitions, hover-эффекты, loading-состояния)

**Не входит:**
- Новые типы заданий (7 существующих остаются) — добавление новых типов заданий выделить в отдельную задачу
- Изменение логики подписки / монетизации (trial/free/active остаётся)
- Изменение backend-авторизации (TMA/JWT/Guest — уже работает)
- Миграция с Preact на другой фреймворк
- Мобильное нативное приложение
- Мультиязычный контент (i18n остаётся ru/en, новые ключи добавляются в обе локали)
- Изменение бота (packages/bot)
- Переработка PvP-дуэлей (остаются как есть)

### ⚠️ ОГРАНИЧЕНИЯ

- **Бюджет бандла:** ≤ 200 КБ gzip для webapp. Radar-chart, анимации и новые стили не должны критично раздуть бандл. Библиотеки графиков (Chart.js, Recharts) слишком тяжёлые — нужно либо SVG вручную, либо легковесную альтернативу (uPlot, canvas-micro)
- **Обратная совместимость:** Telegram Mini App должен продолжать работать. `--tg-theme-*` переменные должны использоваться при наличии, собственная тема — как fallback для веб-версии
- **БД:** Для перцентилей нужна агрегация по `task_attempts`. При большом количестве данных — потенциально тяжёлые запросы. Может потребоваться кэширование в Redis
- **Возрастная группа:** Для сравнения «среди людей вашего возраста» нужно собирать возраст пользователя. Telegram не предоставляет возраст. Нужно запрашивать при онбординге (опционально, можно начать без возраста — сравнивать со всеми)
- **Стек:** Preact (не React) — ограничения по экосистеме компонентов, не все React-библиотеки совместимы
- **Существующие данные:** У текущих пользователей все попытки с `difficulty: 1`. Перцентили будут смещены, пока не наберётся статистика по разным уровням сложности

### 🔍 ФАКТЫ И ПРЕДПОЛОЖЕНИЯ

**Факты** (подтверждённое):
- Генераторы всех 7 заданий уже поддерживают `difficulty: 1-5` с масштабированием параметров
- API `POST /api/tasks/:id/start` принимает `difficulty` в body и валидирует 1-5
- БД хранит `difficulty` в `task_attempts` и `task_sessions`
- UI не передаёт `difficulty` при запуске — используется дефолт `1`
- Страница Progress показывает только streak, дневной бар-чарт и count по категориям
- CSS — один файл `global.css` (~110 строк), без media queries, без responsive breakpoints
- Inline-стили используются повсеместно в компонентах
- Telegram Login Widget рендерится как iframe через script-инъекцию
- Бандл-лимит 200 КБ gzip зафиксирован в архитектуре

**Предположения** (требует проверки):
- Пользователей достаточно для статистически значимых перцентилей (если нет — начать с абсолютных оценок и фиксированных бенчмарков)
- Radar-chart по 4 категориям на SVG можно реализовать в пределах бюджета бандла (~2-5 КБ)
- Пользователи готовы пройти онбординг-тест из 4-8 заданий при первом входе
- Формула расчёта когнитивного рейтинга (нормализованный score по difficulty и типу задания) — рабочая гипотеза, требует валидации
- Возрастная группа может быть необязательной, сравнение по умолчанию — со всеми пользователями

### 📌 ДОПУЩЕНИЯ

- Существующие 7 типов заданий с 5 уровнями сложности дают достаточное разнообразие для удержания пользователя
- Перцентильное сравнение мотивирует лучше абсолютных баллов
- Пользователи веб-версии приходят из десктопных и мобильных браузеров примерно поровну — нужна полная адаптивность
- «Ежедневный челлендж» может определяться серверной логикой без ручной модерации контента
- Для MVP аналитики достаточно 4 когнитивных категорий (memory, attention, logic, speed) — это уже заложено в задачах

### ❓ ОТКРЫТЫЕ ВОПРОСЫ

1. **Откуда брать возраст пользователя для перцентильного сравнения?**
   - Рабочая гипотеза: запрашивать возрастную группу (18-24, 25-34, 35-44, 45+) при онбординге. Опционально — если не указал, сравнивать со всеми.

2. **Какую формулу использовать для когнитивного рейтинга?**
   - Рабочая гипотеза: нормализованный score = `(raw_score / max_possible_score_for_difficulty) × difficulty_weight`. Вес difficulty: 1→1.0, 2→1.5, 3→2.0, 4→2.5, 5→3.0. По каждой категории — средний нормализованный score за последние 30 дней.

3. **Формат ежедневного челленджа — одно задание или серия?**
   - Рабочая гипотеза: одно задание повышенной сложности (difficulty = текущий уровень пользователя + 1, но не выше 5) с 1.5x множителем очков. Серия заданий (мини-тренировка) — отдельная фича.

4. **Как правильно отобразить Telegram Login на веб-версии?**
   - Рабочая гипотеза: вместо встроенного iframe-виджета сделать собственную кнопку «Войти через Telegram» с redirect-flow (Telegram Login Widget redirect mode) или стилизовать контейнер виджета как нативную кнопку приложения с пояснением.

5. **Нужны ли CSS-модули или CSS-фреймворк (Tailwind, UnoCSS)?**
   - Рабочая гипотеза: CSS-модули для компонентных стилей + расширенный `global.css` с CSS-переменными и media queries. Tailwind/UnoCSS добавит зависимость, но может ускорить разработку. Решение — на этапе планирования с учётом бюджета бандла.

6. **Какой объём данных считать достаточным для показа перцентилей?**
   - Рабочая гипотеза: минимум 50 попыток по типу задания × difficulty. До порога — показывать «недостаточно данных» и абсолютный рейтинг по фиксированной шкале.

### 📦 ГОТОВО ДЛЯ ПЛАНИРОВЩИКА

**Тип задачи:** Новая функциональность + рефакторинг UI

**Приоритет (оценка):** Высокий — напрямую влияет на retention и конверсию

**Зависимости:**
- Блок 1 (сложность) и Блок 3 (UI) — независимы, можно параллелить
- Блок 2 (аналитика) зависит от Блока 1 (данные по разным difficulty должны начать накапливаться)
- Онбординг-тест зависит от UI выбора сложности и экрана результатов
- Перцентили зависят от наличия агрегированных данных в БД

**Критерий готовности:**
1. Пользователь может выбрать уровень сложности (1-5) перед запуском любого задания; UI отображает названия уровней
2. После серии заданий пользователь видит свой когнитивный профиль (radar-chart по 4 категориям) с числовым рейтингом
3. Пользователь видит перцентильное сравнение: «Ваш результат выше X% пользователей» (или бенчмарк, если данных мало)
4. Показывается рекомендация: «Чтобы достичь уровня Y%, занимайтесь N месяцев»
5. Есть ежедневный челлендж с повышенной сложностью и бонусом
6. Веб-версия имеет собственную тему (light/dark), визуально привлекательную
7. UI адаптирован под мобильные (320px+), планшеты (768px+), десктоп (1024px+)
8. Вход через Telegram на веб-сайте — интуитивно понятный (собственная кнопка или стилизованный виджет с пояснением)
9. Все новые i18n-ключи добавлены в ru.json и en.json
10. Бандл webapp ≤ 200 КБ gzip
11. Существующая функциональность (TMA, авторизация, дуэли, подписка) не сломана
