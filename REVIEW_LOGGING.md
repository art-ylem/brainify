### 🚨 КРИТИЧНЫЕ ЗАМЕЧАНИЯ (блокируют приёмку)

| # | Файл | Строка/место | Проблема | Что сделать |
|---|------|--------------|----------|-------------|
| 1 | `packages/api/src/db/seed.ts` строка 1 + `packages/api/src/migrate.ts` | `import { db } from './index.js'` в seed.ts | `seed.ts` импортирует `db` из `db/index.ts`, что создаёт **второй** postgres-клиент (CLIENT B). `migrate.ts` создаёт свой собственный клиент (CLIENT A) и закрывает только его (`await client.end()`). CLIENT B остаётся открытым → TCP-соединение держит event loop → `migrate.js` **зависает бесконечно**. Dockerfile CMD: `sh -c "node migrate.js && node index.js"` — из-за `&&` API-сервер никогда не запустится. Контейнер зацикливается на рестартах. | Сделать `seedTasks()` параметризованной: `export async function seedTasks(db: DrizzleInstance)`. В `migrate.ts` передавать тот же `db`, который используется для миграции. Тогда `seed.ts` не импортирует `db/index.ts` и второй клиент не создаётся. Альтернатива: после `seedTasks()` в `migrate.ts` импортировать клиент из `db/index.ts` и закрыть его: `const { db: appDb } = await import('./db/index.js')` → не подходит (клиент не экспортируется). Лучший вариант — параметризация. |
